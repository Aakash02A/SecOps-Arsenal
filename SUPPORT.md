# Support

## Getting Help

If you need help with SecOps-Arsenal, here are the best ways to get support:

### 📖 Documentation

Each tool directory contains its own `README.md` with:
- Feature descriptions
- Prerequisites and installation
- Usage examples
- Safety disclaimers

Start by reading the README of the specific tool you're using.

### 🐛 Bug Reports

If you've found a bug, please open a [Bug Report](../../issues/new?template=bug_report.md) on GitHub. Include:
- Which tool/folder is affected
- Steps to reproduce the issue
- Expected vs. actual behavior
- Your OS and Python version

### 💡 Feature Requests

Have an idea for a new tool or improvement? Open a [Feature Request](../../issues/new?template=feature_request.md).

### 🔒 Security Issues

**Do NOT open a public issue for security vulnerabilities.** Please follow our [Security Policy](SECURITY.md) for responsible disclosure.

### 💬 Discussions

For general questions, ideas, or community conversation, use [GitHub Discussions](../../discussions) (if enabled) or open an issue with the `question` label.

## What We Don't Provide

- **Production support** — these tools are educational and not intended for production security infrastructure.
- **Legal advice** — always consult your organization's legal and compliance teams before conducting security testing.
- **Target-specific assistance** — we cannot help with using tools against systems you do not own or have authorization to test.

## Response Times

This is a community-maintained open-source project. We do our best to respond promptly, but please allow reasonable time for maintainers to review issues and pull requests.
